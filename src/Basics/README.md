# Basics

Fundamentals for HLD/LLD and design patterns: OOP, core Java concepts, and coding practices.

| Topic                          | Folder                                                             | Description                                                                                                                                                      |
| ------------------------------ | ------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **OOP**                        | [OOP/](OOP/)                                                       | Object-oriented programming: class/object, inheritance, polymorphism, encapsulation, abstraction, interfaces, inner classes, `final`, exceptions, best practices |
| **Java concepts**              | [JavaConcepts/](JavaConcepts/)                                      | All Java concepts used in DesignPatterns and SOLID (interfaces, implements, encapsulation, static/final, polymorphism, generics, exceptions, Cloneable, nested classes, DI, fluent API). Simple explanations with examples and links to the repo. |
| **Composition vs inheritance** | [OOP/CompositionVsInheritance.md](OOP/CompositionVsInheritance.md) | When to prefer “has-a” (composition) over “is-a” (inheritance)                                                                                                   |

Code examples in `OOP/` use payment methods (e.g. `Card`, `CreditCard`, `PaymentService`) to illustrate concepts. A runnable demo for several Java concepts is [JavaConcepts/examples/ConceptsDemo.java](JavaConcepts/examples/ConceptsDemo.java).
